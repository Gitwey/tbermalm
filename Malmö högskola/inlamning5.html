“Inline” CSS i jämförelse med externa stilmallar

Man kan applicera CSS på HTML via inline CSS eller en extern stilmall. Inline CSS innebär att man lägger till "style" attributet (style="") direkt i HTML-koden, medan en användningen av en extern stilmall innebär att man har en seperat fil som hanterar all CSS-kod.

Fördelen med inline CSS är att man kan lättare och direkt applicera ändringar på text. Man behöver heller inte ta hänsyn till selektorer (t.ex. id eller class), alltså till vad som koden ska appliceras på (detta görs alltid, om man använder en extern stilmall). Denna fördel kommer automatiskt med en gigantisk nackdel - att man måste applicera CSS på varje enskilt element. Koden blir snabbt rörig och svårt att underhålla. Just denna nackdel är vad en extern stilmall tar hand om.

Om man använder sig av en extern stillmall så behöver man som sagt ange en selektor. Det ända du behöver för att denna selektor ska kunna vara aktuell för en HTML-fil är att ange en referens till CSS-filen. Då kommer automatiskt alla ändringar att tillämpas, ifall de angivna selektorerna finns i HTML-filen. Detta görs i en hierarki (därav namnet: 'Cascading' Style Sheet). En nackdel är dock att man faktiskt måste hämta denna CSS-fil innan ändringarna kan visas, till skillnad från inline CSS som redan finns inbakad i HTML-koden.
